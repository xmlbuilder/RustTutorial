# rem_euclid
- rem_euclid 은 러스트에서 나머지를 항상 **양수 범위** 로 돌려주는 함수.
- 보통 % 연산자는 음수 나머지를 그대로 반환하는데, rem_euclid는 **수학적 유클리드 나머지(Euclidean remainder)** 를 반환합니다.
- 이게 왜 중요하냐면, 각도(normalized angle) 같은 주기적 값 처리할 때 필수적.

## ✅ % 와 rem_euclid의 차이
  - % (Rust의 기본 나머지)
  - 결과가 음수일 수 있음
  - 단순히 **나눗셈의 나머지** 를 반환
- 예:
```rust
println!("{}", -1.0 % 2.0); // -1.0
```


- `rem_euclid`
  - 결과가 항상 0 ≤ r < divisor
  - 수학적 **유클리드 나머지** 정의를 따름
  - 예:
```rust
println!("{}", (-1.0f64).rem_euclid(2.0)); // 1.0
```


## ✅ 왜 각도 계산에서 중요한가?
- 예를 들어 원에서 t % TAU를 하면:
```rust
let t = -0.5;
let a = t % TAU; // -0.5 (음수)
```

- 하지만 각도는 보통 0 ~ 2π 범위로 정규화해야 하죠.
- 이때:
```rust
let a = t.rem_euclid(TAU); // 5.783... (정상적인 양수 각도)
```

## ✅ 직관적 설명
- % → **컴퓨터식 나머지**
- rem_euclid → **수학식 나머지(항상 양수)**

## ✅ 언제 쓰면 좋나?
- 각도 정규화
  - 파라미터 t를 0,1) 또는 0,2π)로 맞출 때
- 주기적 함수
- 원형 인덱스
- 모듈러 연산 기반 알고리즘
- 예:
```rust
let angle = (angle).rem_euclid(2.0 * PI);
```

- 이렇게 하면 angle이 항상 0~2π 범위에 들어갑니다.
