## 📌 왜 기하 피팅을 하는가?
- 데이터 단순화 (모델링)
  - LiDAR, SAR, 레이다 같은 센서 데이터는 수백만 개의 점(포인트 클라우드)으로 이루어져 있습니다.
  - 이걸 그대로 쓰면 너무 방대하고 잡음도 많습니다.
  - 평면, 직선, 원 같은 단순한 수학적 모델로 근사하면 데이터를 압축하고 핵심 구조만 남길 수 있음.
- 잡음 제거 (노이즈 억제)
  - 센서 데이터에는 outlier가 많습니다.
- RANSAC 같은 피팅 알고리즘은 inlier만 남기고 outlier를 제거해서 안정적인 구조를 찾습니다.
- 특징 추출 (Feature Extraction)
  - 도로는 평면, 건물 벽은 직선/평면, 파이프는 원/원통, 나무 줄기는 곡선.
  - 이런 기하 구조를 찾으면 **AI가 인식할 수 있는 특징(feature)** 으로 변환됩니다.
- AI 입력 데이터로 변환
  - 딥러닝 모델(DGCNN, PointNet 등)은 원시 포인트 클라우드도 처리할 수 있지만,  
    기하 피팅으로 **추가적인 특징(예: 평면 법선, 곡률, 반지름)** 을 제공하면 학습 성능이 올라갑니다.
  - 즉, AI는 단순히 점들의 집합보다 구조화된 정보에서 더 잘 학습합니다.

## 🧠 AI와의 연결
- 전처리 단계: AI에 넣기 전에 데이터를 정리하는 과정에서 피팅이 쓰입니다.
  - 예: LiDAR 포인트 → 평면 피팅 → “이건 도로”라는 레이블 생성.
    - 특징 엔지니어링: AI 모델이 스스로 학습하기 어려운 구조를 사람이 먼저 추출해줍니다.
  - 예: 원 피팅 → 반지름 값 → “이건 파이프”라는 특징.
- 하이브리드 접근:
  - 물리 기반 알고리즘(피팅) + 데이터 기반 AI(딥러닝)를 결합하면 성능이 크게 향상됩니다.
  - 예: RANSAC으로 평면 후보를 찾고, CNN으로 “이게 벽인지 도로인지” 분류.

## 📊 예시
- 자율주행: 도로는 평면, 차선은 곡선, 표지판은 원/사각형 → AI가 인식하기 쉽게 구조화.
- 군용 레이다: 표적의 궤적을 직선/곡선으로 피팅 → Kalman/LSTM 추적에 입력.
- 의료 영상: 혈관을 곡선으로 피팅 → AI가 혈관 구조를 더 잘 학습.
- 건축/3D 스캔: 건물 벽을 평면으로 피팅 → CAD/BIM 모델 자동 생성.

## 🔮 결론
- 기하 피팅은 데이터의 의미를 뽑아내는 과정이고,
- AI는 그 의미 있는 구조를 학습·추론하는 데서 큰 도움을 받습니다.
- 즉, 피팅은 AI의 눈을 더 똑똑하게 만들어주는 안경 같은 역할을 합니다.
