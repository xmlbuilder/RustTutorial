# Edge AI
**Edge AI(엣지 인공지능)** 은 데이터를 클라우드나 중앙 서버로 보내지 않고,  
데이터가 생성되는 현장(엣지 디바이스)에서 직접 AI 모델을 실행하는 기술을 말합니다.  
즉, 스마트폰, IoT 센서, 드론, 자율주행차 같은 장치 자체에서 AI가 동작하는 것임.

## 📌 정의
- Edge Computing + AI의 결합
- 데이터가 발생하는 장치(엣지)에서 실시간 분석·추론을 수행
- 클라우드에 의존하지 않고 저지연, 보안성, 효율성을 확보

## 📊 특징
- 저지연(Low Latency): 네트워크 전송 없이 현장에서 바로 처리 → 실시간성 확보
- 프라이버시 보호: 민감한 데이터를 클라우드로 보내지 않아 개인정보 유출 위험 감소
- 네트워크 효율성: 대량의 데이터를 서버로 보내지 않아 대역폭 절약
- 에너지 효율성: 최적화된 모델을 엣지 디바이스에서 실행

## 🎯 활용 사례
- 자율주행차: 차량 내 센서 데이터를 실시간 분석해 즉각적인 제어
- 스마트폰: 얼굴 인식, 음성 비서, 카메라 AI 기능
- 산업 IoT: 공장 센서에서 이상 탐지, 예지 정비
- 헬스케어: 웨어러블 기기에서 심박·혈압 데이터를 실시간 분석
- 스마트 시티: CCTV 영상에서 교통 흐름·안전 상황을 현장에서 분석

## 📌 요약
- 👉 Edge AI는 데이터를 생성하는 장치 자체에서 AI를 실행하는 기술로, 실시간성·보안성·효율성을 강화합니다.
- 👉 클라우드 AI가 "중앙 집중형"이라면, Edge AI는 "현장 분산형"이라고 볼 수 있어요.

# 활용 분야

전투기와 같은 군용 플랫폼에서는 ISAR(역합성 개구 레이더), 표적 감지, 유도형 미사일 시스템에 Edge AI가 매우 중요한 역할을 합니다.  
이유를 단계별로 정리.

## 📌 왜 Edge AI가 필요한가?
- 실시간성
  - 전투 상황에서는 수 밀리초 단위의 반응이 필요합니다.
  - 데이터를 클라우드로 보내 분석하면 지연(latency)이 발생 → 현장에서 바로 처리해야 함.
- 보안성
  - 군사 데이터는 극도로 민감 → 외부 네트워크 전송 없이 장치 내부에서 처리하는 것이 안전.
- 자원 제약
  - 전투기, 미사일, 레이더는 제한된 연산 자원과 전력으로 동작 → 최적화된 Edge AI 모델이 필요.

## 📊 적용 사례
- ISAR (Inverse Synthetic Aperture Radar)
  - 레이더 신호를 실시간으로 처리해 표적의 형상 이미지를 생성.
  - Edge AI는 노이즈 제거, 특징 추출, 표적 분류를 현장에서 수행.
- 표적 감지(Target Detection)
  - EO/IR(전자광학/적외선) 센서 영상에서 적 항공기·지상 표적을 탐지.
  - 딥러닝 기반 객체 탐지 모델을 Edge 디바이스에서 실행해 즉각적인 인식.
- 유도형 미사일(Guided Missile)
  - 미사일 탑재 센서가 표적을 추적하면서 AI가 궤적을 실시간 계산.
  - Edge AI는 비행 중 환경 변화(재밍, 기동, 기상)에 대응해 표적 추적 성능을 유지.

## 🎯 장점
- 지연 최소화 → 즉각적인 대응 가능
- 네트워크 독립성 → 전자전 환경에서도 안정적
- 자율성 강화 → 인간의 개입 없이도 표적 탐지·추적 가능

## 📌 요약
- 👉 전투기와 유도형 무기 체계에서는 ISAR, 표적 감지, 미사일 유도에 Edge AI가 핵심적으로 활용됩니다.
- 👉 이유는 실시간성, 보안성, 자원 제약 때문이며, 이는 군사 AI 시스템의 필수 조건입니다.
